📚 Post API 스펙 테스트

블로그 포스트 API를 테스트하고 스펙을 확인할 수 있습니다

🔗 API 엔드포인트

GET /api/v1/posts

블로그 포스트 목록을 페이지네이션과 함께 조회합니다.

📋 파라미터

파라미터 타입 필수여부 기본값 설명
page Integer 선택 0 페이지 번호 (0부터 시작)
size Integer 선택 10 페이지 크기 (한 페이지당 항목 수)
category String 선택 - 카테고리 필터

🧪 API 테스트

📤 요청 URL:

API 테스트를 실행하면 요청 URL이 여기에 표시됩니다.

📥 응답 결과:

API 테스트를 실행하면 응답이 여기에 표시됩니다.

📖 응답 스키마

{ "content": [ { "postId": 1, "title": "포스트 제목", "category": "기술", "description": "포스트 설명", "url": "https://example.com/post/1", "thumbnailUrl": "https://example.com/thumbnail/1.jpg", "createdBy": "작성자", "createdAt": "2024-01-01T10:00:00" } ], "page": 0, "size": 10, "totalCount": 100 }

🔧 cURL 예제

# 모든 포스트 조회 (기본 페이지네이션) curl -X GET "http://localhost:8080/api/v1/posts" # 특정 카테고리의 포스트 조회 curl -X GET "http://localhost:8080/api/v1/posts?category=기술" # 페이지네이션 사용 curl -X GET "http://localhost:8080/api/v1/posts?page=1&size=20" # 카테고리와 페이지네이션 조합 curl -X GET "http://localhost:8080/api/v1/posts?category=기술&page=0&size=5"
